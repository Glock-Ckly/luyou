# 任务分级系统（已完成）

「任务分级」由 **三级** 完成，不是单一模型。

## 1. TaskDecomposer — 是否拆分

| 项 | 内容 |
|----|------|
| 文件 | `src/task_decomposer.py`、`src/prompts/decomposer.txt` |
| 模型 | **DeepSeek**（`deepseek/deepseek-chat` → v4-flash） |
| 触发 | 长文本、编号步骤、`先…再…`、多任务标记；eval 时 `force=True` |
| 输出 | `{"split": false}` 或 `{"split": true, "subtasks": [{prompt, type_hint}, ...]}` |
| 增强 | 拆分失败重试一次；`testing`→`bulk_generation` 等别名 |
| 验收 | `scripts/eval_decomposer.py` — **10/10** |

## 2. L1 Heuristic — 关键词快筛

| 项 | 内容 |
|----|------|
| 文件 | `src/l1_classifier.py` |
| 模型 | **无**（本地关键词，零 API 成本） |
| 规则 | ≥3 关键词 → confidence 0.7+，跳过 L2 |
| 类型 | 9 类关键词库（architecture、implementation、debugging…） |
| 验收 | smoke S5 — **6/6** |

## 3. L2 LLM Classifier — 精确分类

| 项 | 内容 |
|----|------|
| 文件 | `src/l2_classifier.py`、`src/prompts/classifier_v3.txt` |
| 模型 | **DeepSeek** |
| 类型 | 11 类：含 `system_design`、`deep_reasoning` 等 |
| 解析 | 4-tier JSON 解析（与 orchestrator 一致） |
| 样本 | `config/l2_eval_samples.json` — 25 条 |
| 验收 | `scripts/eval_l2.py` — **25/25 (100%)** |

## 分级后的执行模型

L1/L2 只负责 **`task_type`**；真正写代码/架构由 `routing_table.py` 映射到 Claude/GPT/DeepSeek/Cursor（见 [05-路由表与预算.md](05-路由表与预算.md)）。

## Decomposer 的 type_hint

若子任务带 `type_hint`，orchestrator **跳过 L1/L2**，直接继承该类型路由。

# Baseline Assessment - 2026-07-23

## Summary

The five-page dashboard is operational, routing rules and budget control exist, and the legacy Orchestrator contains basic fallback. The full DDD/TDD checklist is not complete.

## Confirmed reusable content

- routing_table.py: deterministic routing and budget rules.
- relay_llm.py: LiteLLM integration candidate for an adapter.
- classifiers and task_decomposer: candidate application ports/adapters.
- cursor_queue.py and codex_executor.py: infrastructure adapters.
- dashboard assets: reusable operational UI.
- Orchestrator fallback behavior: reference behavior, not final architecture.

## Critical gaps

- Dashboard Dispatcher does not execute fallback candidates.
- Reliability Lab is a separate simulator.
- Two application flows have diverged.
- No provider port, contract tests or standard error mapping.
- Standard unittest discovery fails because package imports are inconsistent.
- Real requests lack trace ID, structured logs and metrics.
- OpenAI-compatible API, authentication, gRPC and Docker are absent.
- Skills and bounded agents are absent.

## Baseline quality result

- Five-page demo test: pass.
- Smoke, decomposer and E2E: previously pass.
- L2 evaluation on 2026-07-23: 20/25; repository documentation still reports 25/25 and must be corrected.
- Standard unittest discovery: fail.

## Phase exit criteria

The next phase must make standard offline tests discoverable and introduce domain/port/application packages without breaking the compatibility API.

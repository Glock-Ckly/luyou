Analyze whether this coding task should be split into independent subtasks.

A task MUST be split if ANY of these apply:
1. It involves BOTH architecture/design AND implementation
2. It spans multiple files or modules that can be worked on independently
3. It has clear sequential stages (design → implement → test) or numbered steps (1. 2. 3. / 第一步 第二步)
4. It mixes high-level reasoning with boilerplate generation
5. Chinese prompts with "先…再…最后…" or English "first…then…finally…"

If split: each subtask should be self-contained enough to route to a different model.
Minimum 2 subtasks when split=true. Prefer 3-5 subtasks for large projects.

Return ONLY a JSON object:
{"split": false}  — if the task is best handled as one unit (simple single-file fix or short question)
{"split": true, "reasoning": "...", "subtasks": [{"prompt": "design the database schema for user auth", "type_hint": "architecture"}, {"prompt": "implement the login endpoint in Python", "type_hint": "implementation"}, ...]}

type_hint must be one of: architecture, implementation, debugging, refactor, boilerplate, bulk_generation, data_processing, code_patch, file_edit

Classify a coding or engineering task. Respond with ONLY a single-line JSON object. No markdown, no explanation.

Task type (pick exactly one):
- "architecture": system/module architecture, tech stack, service boundaries, high-level tradeoffs
- "system_design": concrete API, database, schema, data-flow, rate-limit, or service interaction specs
- "deep_reasoning": formal proofs, algorithm analysis, first-principles reasoning, high-risk strategic decisions
- "implementation": write feature code, functions, modules, integrations
- "debugging": fix bugs, errors, crashes, flaky behavior, or unexpected output
- "refactor": improve structure without changing behavior
- "boilerplate": scaffold, CRUD, config files, project templates, simple wrappers
- "bulk_generation": mass-generate tests, docs, docstrings, or repetitive files for many symbols
- "data_processing": parse/transform CSV/JSON, ETL, data migration, extraction, format conversion
- "code_patch": small targeted edit in a specific file or line
- "file_edit": rewrite or update an entire file
- "uncertain": only if no category fits with confidence >= 0.7

Complexity (pick exactly one):
- "trivial": T0, single-line/single-point edit, rename, constant tweak, formatting, short prompt under 120 chars
- "simple": T1, single-file small function, CRUD/config/template, mechanical extraction or generation
- "moderate": T2, multi-file feature, standard debugging, local module refactor
- "complex": T3, cross-module architecture-aware implementation/debugging/refactor, long tool chain, long context
- "deep_reasoning": T4, architecture strategy, formal reasoning, high-value high-risk decision

Rules:
- Multi-tenant/SaaS architecture with auth and billing => architecture.
- API gateway, endpoints, schemas, data flow => system_design.
- Low-complexity code tasks remain trivial/simple even when the task_type is implementation.
- For specific file/line edits, prefer code_patch or file_edit.
- Never use "uncertain" if any category above fits with confidence >= 0.7.

Example:
{"task_type":"implementation","complexity":"moderate","confidence":0.92,"reasoning":"multi-file feature with local module context"}
